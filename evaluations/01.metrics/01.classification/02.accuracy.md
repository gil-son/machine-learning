# Accuracy

## <td align="center"><img src="https://cdn-icons-png.flaticon.com/512/7963/7963858.png" width="80"/> Introduction

Accuracy is one of the most common evaluation metrics used in classification problems.

It measures **how many predictions the model got right out of all predictions made**.

In other words:

Accuracy answers the question:
**"Out of all predictions, how many were correct?"**

It is calculated using the Confusion Matrix values:

- True Positive (TP)
- True Negative (TN)
- False Positive (FP)
- False Negative (FN)

### Formula

Accuracy = (TP + TN) / (TP + TN + FP + FN)

This means:
- Correct predictions → TP + TN
- Total predictions → TP + TN + FP + FN

---
 
## <td align="center"><img src="https://cdn-icons-png.flaticon.com/512/7527/7527144.png" width="80"/> How it works?

```
Dataset
   ↓
Train / Validation Split
   ↓
Model Training
   ↓
Predictions
   ↓
Evaluation Metrics (Accuracy)
   ↓
Model Selection
```

### Step-by-step Example

1. Dataset

Email spam classification dataset:

| Email          | Label    |
| -------------- | -------- |
| Win money now  | Spam     |
| Meeting at 3pm | Not Spam |
| Free vacation  | Spam     |
| Project update | Not Spam |

---

2. Train / Validation Split

Train: 80%  
Validation: 20%

- Training set → used to train the model  
- Validation set → used to evaluate performance  

---

3. Model Training

Train a classifier:

Examples:
- Logistic Regression
- Decision Tree
- Random Forest

The model learns patterns like:

- "win money" → spam  
- "free vacation" → spam  
- "meeting" → not spam  

---

4. Predictions

| Email          | Actual   | Predicted |
| -------------- | -------- | --------- |
| Win money now  | Spam     | Spam      |
| Meeting at 3pm | Not Spam | Spam      |
| Free vacation  | Spam     | Spam      |
| Project update | Not Spam | Not Spam  |

---

5. Confusion Matrix Values

- TP = 2  
- TN = 1  
- FP = 1  
- FN = 0  

---

6. Accuracy Calculation

Accuracy = (TP + TN) / (TP + TN + FP + FN)

Accuracy = (2 + 1) / (2 + 1 + 1 + 0)  
Accuracy = 3 / 4  
Accuracy = 0.75 → **75%**

---

## <td align="center"><img src="https://cdn-icons-png.flaticon.com/512/5557/5557844.png" width="80"/> Why use it?

1. Simple and intuitive

Accuracy is easy to understand:
- Higher accuracy → better model (in general)

---

2. Good for balanced datasets

When classes are evenly distributed, accuracy gives a reliable measure.

Example:
- 50% Spam
- 50% Not Spam

---

3. Quick model comparison

You can quickly compare multiple models using accuracy.

Example:
- Model A → 80%
- Model B → 85% → better (in most cases)

---

4. Useful as a baseline metric

Accuracy is often the **first metric** to check before analyzing more advanced metrics.

---

## <img src="https://cdn-icons-png.flaticon.com/512/6675/6675847.png" width="80"> Limitations of Accuracy

Accuracy can be misleading in **imbalanced datasets**.


Example:

- 95% Not Fraud
- 5% Fraud

A model that predicts "Not Fraud" for everything:

Accuracy = 95%  
But it completely fails to detect fraud.

This is why metrics like:
- Precision
- Recall
- F1 Score

are also important.

## <img src="https://cdn-icons-png.flaticon.com/512/8633/8633476.png" width="80"> Tips of Accuracy

### 1. Don’t use Accuracy alone

Accuracy is a good starting point, but **never rely only on it**.

Always combine with:
- Precision
- Recall
- F1 Score

>Especially when the cost of errors matters.

---

### 2. Be careful with imbalanced datasets

If one class dominates the dataset, accuracy can be misleading.

Example:
- 99% Not Fraud
- 1% Fraud

A model that always predicts *Not Fraud* → **99% accuracy**, but useless.

> Always check the Confusion Matrix.

---

### 3. Understand the type of error

Accuracy does not tell you:
- How many **false positives**
- How many **false negatives**

Ask:
- Is missing a positive case worse?
- Or triggering false alarms worse?

---

### 4. Use Accuracy for quick comparisons

Accuracy is useful to:
- Compare baseline models
- Track early improvements

>But for final decisions → use more detailed metrics.

---

### 5. Normalize when comparing datasets

Accuracy only makes sense when:
- Datasets are similar
- Class distributions are comparable

>Otherwise, comparisons may be misleading.

---

### 6. Always relate Accuracy to the problem

Different problems require different priorities:

- Medical diagnosis → prioritize **Recall**
- Spam detection → balance **Precision & Recall**
- General classification → Accuracy can be useful

> Accuracy is not “wrong” — it’s just **not enough alone**.

---

### 7. Track Accuracy over time

During training:
- Monitor training vs validation accuracy
- Detect **overfitting**

Example:
- Training accuracy ↑
- Validation accuracy ↓

> Model is memorizing, not generalizing

## <td align="center"><img src="https://cdn-icons-png.flaticon.com/512/6404/6404117.png" width="80"/> Summary

Accuracy answers a simple but important question:

**How often is the model correct?**

Key points:

- Based on Confusion Matrix values (TP, TN, FP, FN)
- Easy to compute and interpret
- Works well for balanced datasets
- Can be misleading for imbalanced data

Accuracy is a great starting point, but should always be combined with other metrics for a complete evaluation.
