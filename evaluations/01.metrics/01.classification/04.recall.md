# Recall

## <td align="center"><img src="https://cdn-icons-png.flaticon.com/512/7963/7963858.png" width="80"/> Introduction

Recall is a classification metric that measures **how many actual positives were correctly identified by the model**.

It answers the question:

**"Out of all real positives, how many did the model find?"**

Recall focuses on **False Negatives**.  
It is especially important when **missing a positive case is costly**.

Recall is calculated using the Confusion Matrix values:

- True Positive (TP)
- False Negative (FN)

### Formula

Recall = TP / (TP + FN)

This means:

- Correct positive predictions → TP  
- Total actual positives → TP + FN  

High Recall → Few False Negatives  
Low Recall → Many missed positives  

---

### When Recall Matters

Recall is important when **missing a positive case is dangerous**.

Examples:

- Fraud detection → don't miss fraud transactions  
- Medical diagnosis → don't miss sick patients  
- Spam detection → don't miss spam emails  
- Anomaly detection → don't miss critical alerts  

---

## <td align="center"><img src="https://cdn-icons-png.flaticon.com/512/7527/7527144.png" width="80"/> How it works?

```
Dataset
   ↓
Train / Validation Split
   ↓
Model Training
   ↓
Predictions
   ↓
Evaluation Metrics (Recall)
   ↓
Model Selection
```

### Step-by-step Example

1. Dataset

Email spam classification dataset:

| Email          | Label    |
| -------------- | -------- |
| Win money now  | Spam     |
| Meeting at 3pm | Not Spam |
| Free vacation  | Spam     |
| Project update | Not Spam |

---

2. Train / Validation Split

Train: 80%  
Validation: 20%

- Training set → used to train the model  
- Validation set → used to evaluate performance  

---

3. Model Training

Train a classifier:

Examples:
- Logistic Regression
- Decision Tree
- Random Forest

The model learns patterns like:

- "win money" → spam  
- "free vacation" → spam  
- "meeting" → not spam  

---

4. Predictions

| Email          | Actual   | Predicted |
| -------------- | -------- | --------- |
| Win money now  | Spam     | Spam      |
| Meeting at 3pm | Not Spam | Spam      |
| Free vacation  | Spam     | Spam      |
| Project update | Not Spam | Not Spam  |

---

5. Confusion Matrix Values

- TP = 2  
- TN = 1  
- FP = 1  
- FN = 0  

---

6. Recall Calculation

Recall = TP / (TP + FN)

Recall = 2 / (2 + 0)  
Recall = 2 / 2  
Recall = 1.0 → **100%**

---

Interpretation:

There were **2 real spam emails**  
The model correctly detected **both**

Recall = **100%**

This means:
- No spam was missed
- All positive cases were found

---

## <td align="center"><img src="https://cdn-icons-png.flaticon.com/512/5557/5557844.png" width="80"/> Why use it?

### 1. Reduces missed positives

Recall measures how many real positives were detected.

High recall → fewer missed cases

Example:
Fraud detection missing fraud is dangerous.

---

### 2. Important when missing positives is costly

Use recall when False Negatives are worse than False Positives.

Examples:

- Disease detection → missing sick patient is dangerous  
- Fraud detection → missing fraud costs money  
- Security alerts → missing attack is critical  

---

### 3. Measures detection ability

Recall tells how good the model is at **finding positives**.

Example:

Model A:
Recall = 95%

Model B:
Recall = 60%

Model A finds **more real positives**

---

### 4. Works together with Precision

Recall alone is not enough.

You should analyze together:

- Precision → false positives  
- Recall → false negatives  

This leads to:
- F1 Score
- Precision-Recall curve

---

### 5. Useful for imbalanced datasets

When positive class is rare:

- Fraud detection  
- Disease detection  
- Anomaly detection  

Recall becomes very important.

---

## <img src="https://cdn-icons-png.flaticon.com/512/8228/8228687.png" width="80"> Recall vs Precision

Recall and Precision measure different things.

### Precision

Precision answers:

**"When the model predicts positive, how often is it correct?"**

Precision = TP / (TP + FP)

Focus:
False Positives

---

### Recall

Recall answers:

**"Out of all real positives, how many were found?"**

Recall = TP / (TP + FN)

Focus:
False Negatives

---

### Example

|                 | Pred Spam | Pred Not Spam |
|-----------------|-----------|---------------|
| Actual Spam     | 2 (TP)    | 0 (FN)        |
| Actual Not Spam | 1 (FP)    | 1 (TN)        |

Precision:

Precision = 2 / (2 + 1) = 66%

Recall:

Recall = 2 / (2 + 0) = 100%

---

### Key Difference

Precision → cares about **false positives**  
Recall → cares about **false negatives**

---

### When to prioritize Precision

- Spam filter  
- Search results  
- Recommendation systems  
- Email classification  

Goal: avoid false alarms

---

### When to prioritize Recall

- Medical diagnosis  
- Fraud detection  
- Security alerts  
- Defect detection  

Goal: avoid missing positives
